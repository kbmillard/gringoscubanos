Build / update the La Hamburguesa Loca one-page website as a luxury, interactive, online-ordering homepage inspired by the interaction style of https://www.masgirbau.com/en, but do not copy its text, images, layout exactly, or assets.

The goal is a one-page online ordering experience where every button opens, reveals, swaps, or expands content on the homepage instead of navigating to separate pages.

- Next.js App Router
- TypeScript
- Tailwind CSS
- Framer Motion
- Lucide icons
- shadcn/ui if useful
- Mobile-first responsive design

Overall design direction:
Create a cinematic, editorial, hospitality-style restaurant website with the calm premium interaction feel of Mas Girbau, adapted for a Mexican restaurant / food truck / online ordering business.

Visual style:
- Full-screen hero
- Dark cinematic image overlays
- Warm colors: charcoal, cream, tortilla beige, salsa red, grilled orange, muted green
- Deep purple / plum menu background inspired by the provided menu image
- Cyan for Hamburguesas
- Green for Tacos
- Yellow for Tortas
- Hot pink for Antojitos
- Orange for Kids Menu
- Red for Drinks
- Thin uppercase navigation
- Large serif or editorial-style headlines
- Smooth section transitions
- Numbered content blocks
- Big image panels
- Soft hover reveals
- Dotted divider lines between item names and “Price TBD” when price is null
- Playful Mexican food-truck energy, but still polished and premium
- No clutter

Core rule:
Do not build a normal restaurant website. Build the homepage like an interactive ordering room. Every click should make the front page change shape. No dead CTAs. No separate menu pages.

GOOGLE SHEET MENU MANAGEMENT SYSTEM

Add an easy menu-management system so restaurant staff can update the menu from a Google Sheet without touching code, GitHub, Vercel, or Cursor.

Implement a menu data adapter with two sources:
1. Google Sheets CSV source
2. Local fallback source using the menuItems array above

Use the Google Sheet as the primary source when NEXT_PUBLIC_MENU_CSV_URL or MENU_CSV_URL exists.
If the Google Sheet fetch fails, fall back to the local TypeScript menu data.

Create this structure:

/lib/menu/schema.ts
/lib/menu/local-menu.ts
/lib/menu/google-sheet-menu.ts
/lib/menu/get-menu.ts
/app/api/menu/route.ts

Google Sheet columns:
active
category
sortOrder
name
description
price
includesFries
meatChoiceRequired
featured
imageUrl

Optional Google Sheet column:
id

Parsing rules:
- active accepts TRUE/FALSE, true/false, yes/no, 1/0
- price can be blank; blank becomes null
- includesFries, meatChoiceRequired, and featured should parse as booleans
- id should be generated from category + name if no id column exists
- inactive rows should not display on the public menu
- sort menu items by category, then sortOrder, then name
- trim whitespace from all cell values
- gracefully ignore fully blank rows
- keep category names exactly as staff enter them, but normalize known categories to the menuCategories order when possible

API:
Create /api/menu that returns:

{
  categories: string[];
  items: MenuItem[];
  featuredItems: MenuItem[];
  source: "google-sheet" | "local-fallback";
  updatedAt: string;
}

Caching:
- Use Next.js fetch caching with revalidate: 300
- This means the menu can update roughly every 5 minutes without redeploying
- Add a comment explaining how to reduce this later if needed

Frontend:
- Replace direct imports of local menuItems with a client-side fetch to /api/menu
- Show a loading skeleton while menu loads
- If source is local-fallback, optionally show a small hidden developer console warning only in development mode
- Keep all existing interactions:
  - category switching
  - add to cart
  - meat choice selector
  - cart drawer
  - featured/popular orders
  - price TBD behavior when price is null

Google Sheet owner workflow:
Add a README section explaining how the restaurant updates the menu:

1. Open the Google Sheet
2. Add/edit item rows
3. Set active to TRUE or FALSE
4. Add prices when confirmed
5. Wait up to 5 minutes for the site to refresh

Do not build a full admin dashboard yet.
Use Google Sheets first because it is easiest for a restaurant owner to maintain.
Keep the code modular so a future Prisma/admin dashboard can replace Google Sheets later.
